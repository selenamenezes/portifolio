Sobre

Como eu descrevo projetos de dados sem dashboard

O que mostrar no portfólio

  • Entrada: origem dos dados (APIs, arquivos, banco, logs)
  • Pipeline: etapas ETL/ELT e orquestração
  • Qualidade: validações, checagens, testes e tratamento de erros
  • Entrega: outputs (tabelas/partições/arquivos) e como usar
  • Reprodutibilidade: instruções para rodar (README)

Narrativa pronta (pra você copiar)

“Construí um pipeline reprodutível para transformar dados brutos em dados confiáveis, garantindo qualidade e rastreabilidade do começo ao fim.”

A cada projeto, destaque o impacto: tempo reduzido, menos falhas, padronização, custo menor, ou maior confiabilidade.

Dica: inclua prints só do que for necessário — logs e validações contam muito.

Como rodar (substitua)

  1. Instale dependências: pip install -r requirements.txt
  2. Configure variáveis em .env
